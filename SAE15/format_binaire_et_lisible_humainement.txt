Un format binaire est un format de données qui est enregistré sous 
forme de bits (0 et 1) sur un support de stockage, comme un disque 
dur ou une clé USB. Ces données sont généralement difficiles à lire 
et à comprendre pour les humains, car elles ne sont pas présentées 
sous une forme lisible directement.

Un format lisible humanement, en revanche, est un format de données 
qui est présenté sous une forme qui peut être lue et comprise par 
les humains, généralement en utilisant un langage naturel ou une 
syntaxe spécifique. Ces formats sont généralement plus faciles à 
utiliser et à manipuler pour les humains, mais peuvent être moins 
efficaces en termes de stockage et de transmission de données.

Voici deux exemples de formats lisibles humainement :

1- Le format JSON (JavaScript Object Notation) est un format de 
   données structuré qui est souvent utilisé pour stocker et échanger 
   des données sur Internet. Il est basé sur la syntaxe du langage de 
   programmation JavaScript, mais peut être utilisé avec de nombreux 
   autres langages de programmation.

2- Le format XML (eXtensible Markup Language) est un format de données 
   structuré qui est souvent utilisé pour stocker et échanger des 
   données en ligne. Il utilise des balises pour encoder des 
   informations et des données, de manière à ce qu'elles soient 
   lisibles et compréhensibles par les humains.

Il existe de nombreux autres formats lisibles humanement, tels que CSV 
(Comma Separated Values), HTML (Hypertext Markup Language), ou encore 
YAML (YAML Ain't Markup Language). Chacun de ces formats a ses propres 
caractéristiques et utilisations spécifiques.
